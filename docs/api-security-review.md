## Step 2: Authentication Review

### JWT Implementation

| Check | Status | Detail |
|---|---|---|
| Algorithm pinned server-side | ❌ FAIL | `algorithms=None` — accepts any alg including `"none"` (`auth.py:113`) |
| Expiry enforced | ❌ FAIL | No `exp` claim issued; `verify_exp: False` in decode (`auth.py:114`) |
| Secret strength | ❌ FAIL | `JWT_SECRET = "secret"` — 6 characters, trivially brute-forceable (`auth.py:22`) |
| Secret from environment | ❌ FAIL | Hardcoded in source; duplicated as `SECRET_KEY` in `main.py:35` |
| Signature verification | ❌ FAIL | `alg=none` header bypasses HMAC entirely |
| Token revocation | ❌ FAIL | No blocklist or session store; tokens unrevocable once issued |
| Secure transmission | ❓ UNKNOWN | No TLS enforcement; uvicorn launched without SSL config |

### Password Storage

| Check | Status | Detail |
|---|---|---|
| Algorithm | ❌ FAIL | MD5 — general-purpose hash with no work factor (`auth.py:29,81,162`) |
| Salting | ❌ FAIL | No per-user salt; identical passwords produce identical hashes |
| Admin hash | ❌ FAIL | `md5("admin123")` cracks instantly via rainbow table |

### Token Forge / Steal Vectors

Six independent paths to a valid token:

1. **`alg=none` forgery** — `jwt.decode(algorithms=None)` accepts unsigned tokens (`auth.py:110-113`)
2. **Offline brute-force** — secret `"secret"` cracks in milliseconds with hashcat
3. **`GET /admin/debug/tokens`** — returns valid JWTs for every user with zero authentication
4. **`GET /health`** — returns `SECRET_KEY` to any unauthenticated caller; enables offline token signing
5. **Path traversal** — `GET /files/download?filename=../../app/auth.py` exposes `JWT_SECRET`
6. **Predictable reset token** — `MD5(username + floor(unix_ts/60))` brute-forced in < 60 seconds

## Step 5: API-Specific Vulnerabilities

### Mass Assignment

`PUT /admin/user/{user_id}` exposes `role` as a writable field for any authenticated user:

```python
class UpdateUserRequest(BaseModel):
    username: Optional[str] = None
    role: Optional[str] = None    # user should NEVER be able to set this
    password: Optional[str] = None
```

Any user submitting `{"role": "admin"}` self-promotes in a single request.

### BOLA / IDOR

Four IDOR vulnerabilities with the same root cause — no ownership assertion after authentication:

| Endpoint | Missing Check |
|---|---|
| `GET /admin/user/{user_id}` | `current_user["id"] != user_id AND role != "admin"` |
| `PUT /admin/user/{user_id}` | Same as above |
| `POST /data/transfer` | `account.owner == current_user["sub"]` |
| `DELETE /files/delete` | File owner tracking entirely absent |

The `/admin/user/{user_id}` IDOR exposes `password_hash` — MD5 hashes that crack instantly via rainbow tables.

### Excessive Data Exposure

| Endpoint | Should Return | Actually Returns |
|---|---|---|
| `GET /admin/user/{id}` | `id`, `username`, `role` | + **`password_hash`** |
| `GET /admin/panel` | Sanitized user list | Full `USERS_DB` with all **`password_hash`** values |
| `GET /auth/forgot-password` | Nothing | **`reset_token`** + **`hint` (algorithm disclosure)** |
| `GET /health` | `{"status":"ok"}` | **`SECRET_KEY`** + **`dict(os.environ)`** |
| HTTP 500 | `{"error":"Internal server error"}` | **`traceback`**, **`internal_path`**, **`db_password`** |

### Missing Rate Limits

Every endpoint has no rate limiting. Highest-impact absences:

| Endpoint | Attack Enabled |
|---|---|
| `POST /auth/login` | Unlimited password brute-force |
| `POST /auth/forgot-password` | Reset token timing attack |
| `GET /fetch` | SSRF amplification / server-as-proxy DDoS |
| `GET /data/account` | SQL injection probing with no detection |

> The dead rate-limiting code in `auth.py:69-72` has threshold `> 1000` and body `pass` — a particularly misleading false safeguard.

## Step 7: HTTP Security Headers

None of the following headers are configured:

| Header | Impact of Absence |
|---|---|
| `Content-Security-Policy` | XSS executes on `/docs` and `/redoc` |
| `X-Content-Type-Options: nosniff` | Browser MIME-sniffs uploaded files — uploaded HTML/JS served as scripts |
| `X-Frame-Options: DENY` | API responses and Swagger UI can be framed; enables clickjacking |
| `Strict-Transport-Security` | Clients not forced to HTTPS; Bearer tokens interceptable |
| `Cache-Control: no-store` | Auth tokens cached by proxies and browsers |
| `Referrer-Policy` | Bearer tokens in URLs leak via `Referer` header |

**Most critical:** `X-Content-Type-Options: nosniff` — without it, unrestricted file upload becomes a stored XSS vector when HTML/SVG files are served inline.

**Swagger UI** (`/docs`, `/redoc`) is publicly accessible — the full interactive API schema is available to any visitor.

### Security Headers Middleware
```python
from starlette.middleware.base import BaseHTTPMiddleware

class SecurityHeadersMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        response = await call_next(request)
        response.headers["X-Content-Type-Options"] = "nosniff"
        response.headers["X-Frame-Options"] = "DENY"
        response.headers["Cache-Control"] = "no-store"
        response.headers["Referrer-Policy"] = "no-referrer"
        return response

app.add_middleware(SecurityHeadersMiddleware)
```

### Password Hashing
```python
# Before — auth.py:81
password_hash = hashlib.md5(password.encode()).hexdigest()

# After
import bcrypt
password_hash = bcrypt.hashpw(password.encode(), bcrypt.gensalt()).decode()

def verify_password(plain: str, stored_hash: str) -> bool:
    return bcrypt.checkpw(plain.encode(), stored_hash.encode())
```
